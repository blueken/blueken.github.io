fetal-movement-deploy

目标

使用 serverless Framework 将前后端分别上线。

本期的任务很简单，但是需要处理的细节很多，请系好安全带，飞机要起飞了。

后端 deploy

➜ piglet-back git:(master) serverless deploy

Serverless: Stack update finished…

service: piglet-back

stage: dev

region: us-east-1

stack: piglet-back-dev

endpoints:
GET - > https://6xj4j9nu7j.execute-api.us-east-1.amazonaws.com/dev/fms

POST - > https://6xj4j9nu7j.execute-api.us-east-1.amazonaws.com/dev/fms

functions:

getAll: piglet-back-dev-getAll

addOne: piglet-back-dev-addOne

前端

将 API 相关接口从 localhost:3000/fms改为后端serverless 部署的URL

vi FChart.js

vi Form.js

const URL = ‘https://6xj4j9nu7j.execute-api.us-east-1.amazonaws.com/dev/fms'

gatsby server

好了，浏览器访问 http://localhost:8080 不出所料， http 页面调用 https 接口是不通的。

localhost 添加 https

找到了 medium的相关文章

还找到了 letsencrypt相关文章

甚至还有 github相关工具

但是， chotto ma tte， gatsby 要是有支持https就好了，去官网睢一眼

https://www.gatsbyjs.org/docs/local-https/

很好一个参数 –https or -S 搞定

$ gatsby develop –https

访问 https://localhost:8000/

太好了，页面出来了。

等下，还有问题

页面中数据还是没有。 F12 看下：

原来 https://6xj4j9nu7j.execute-api.us-east-1.amazonaws.com/dev/fms 报错了：

access to XMLHttpRequest at ‘https://6xj4j9nu7j.execute-api.us-east-1.amazonaws.com/dev/fms' from origin ‘https://localhost:8000' has been blocked by CORS policy: No ‘Access-Control-Allow-Origin’ header is present on the requested resource.

解决Sercerless API CORS 问题

  1. step1

    参考https://www.altcademy.com/questions/amazon-aws-s3-error-no-access-control-allow-origin-header-is-present-on-the-requested-resource

https://console.aws.amazon.com/s3 -> piglet-back-dev-serverlessdeploymentbucket-1tbvuksuf7cct  -> permissions -> CORS configuration



保存后，发现问题依旧

  1. step2

    参考 : https://stackoverflow.com/questions/35190615/api-gateway-cors-no-access-control-allow-origin-header

> vi piglet-back/handler.js



step1 是否是 step2 的必须前置依赖，笔者没有尝试，读者可直接使用 step2 来尝试解决问题。

  1. step3

    打开页面，终于可以GET拿到数据了, 但是还有问题， POST 数据时还是报 No ‘Access-Control-Allow-Origin’ header is present， 即使给 POST 对应的接口添加 headers 也不行

    vi piglet-back/handler.js

    想到 serverless 不可能 不支持CORS 的，去官网一看，果然有方案:

    https://serverless.com/framework/docs/providers/aws/events/apigateway#enabling-cors

    vi piglet-back/serverless.yml

    加上 cors: true 搞定。

    PS: 前面 getAll 接口是不是也可能直接 通过 step3 这一个 cors: true 来解决，笔者也没有尝试，读者可以尝试一下。
