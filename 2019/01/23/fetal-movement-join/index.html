小猪胎动联调篇

目标

前端，后端 独立应用已经 ready, 接下来进行数据接入. Gatsby 有自己的数据接入层（GraphQL），考虑到应用本身比较 tiny ，我们放弃使用数据接入层，使用 axios 来直接调用 serverless api

安装 axios

npm i -S axios

图表拉取数据

vi src/components/FCharts.js

表格提交数据

vi src/components/Form.js

addOne 接口使用真实数据

vi handler.js

运行效果

piglet-back git:(master) ✗ serverless offline start

piglet-front git:(master) ✗ gatsby develop

visit http://localhost:8000/

遇到一些问题？

Q1: 请求接口时后台报错：[ ‘OverwriteModelError: Cannot overwrite FmModel model once compiled.’,

A1: serverless offline start 增加 –useSeparateProcesses 或者 –skipCacheInvalidation

参考： https://github.com/dherault/serverless-offline/issues/258

延伸阅读

gatsby-node.js

创建后端路由

vi gatsby-node.js

touch src/components/XXX.js

gatsby develop

visit http://localhost:8000/xxx

浏览器中显示： XXX: bobalan

gatsby-browser.js

可以用来处理全局样式

gatsby-config.js

gatsy 配置文件，包括数据和插件

gatsby-ssr.js

gatsy 配置ssr

番外篇: 前后端路由同名优先级？

根据推断有准确后端路由的会渲染对应的组件，只有没有命中后端路由，才会交给前端路由。让我们来实际测试一下。

Gatsby starter 应用中默认有一个 src/pages/page-2的前端路由，让我们把后端路由也改在 page-2

vi gatsby-node.js

gatsby develop

visit http://localhost:8000/page-2

浏览器中显示： XXX: bobalan

验证我们的推测正确
